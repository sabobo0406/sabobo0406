# Shokz イヤホンで AI にハンズフリー音声入力するセットアップ

「キーボードを押しながら話す」のではなく、**手元のボタン(または Shokz 本体のボタン)を押すと、Shokz のマイクから音声入力が始まる**仕組みを作るためのガイドです。

結論: **できます。** 方法は大きく3つあり、組み合わせも可能です。

---

## ⚡ クイックスタート(方法A・Typeless / Mac)

Mac のターミナルに**この1行を貼るだけ**で、Hammerspoon のインストール〜設定ファイル配置〜起動まで自動で行います:

```bash
curl -fsSL https://raw.githubusercontent.com/sabobo0406/sabobo0406/claude/shokz-earbud-voice-input-ycnjy4/shokz-voice-input/install.sh | bash
```

実行後、画面に出る残り3ステップ(① Hammerspoon の権限許可 ② Typeless のホットキーを `⌘⌥D`・トグルに設定 ③ マイクを Shokz に選択)を行えば完了です。
手動で1つずつ進めたい場合は、下の各方法の手順を参照してください。

---

## 方法A: 追加ハードなし — Shokz 本体のマルチファンクションボタンを使う(Mac)

Shokz の再生/一時停止ボタンは、Mac には「メディアキー」として届きます。
[Hammerspoon](https://www.hammerspoon.org/)(無料)でこのイベントを横取りして、「音楽の再生/停止」の代わりに「音声入力の開始/停止」に割り当てます。

- メリット: 何も買わなくていい。ボタンは常に耳元にあるので、部屋の中を歩き回りながら使える(Bluetooth が届く約10mの範囲)。
- デメリット: 「押している間だけ」ではなく「1回押すと開始、もう1回押すと終了」のトグル式になる(ヘッドセットのボタンは長押しが別機能に割り当てられているため)。

### セットアップ手順

1. Hammerspoon をインストール: `brew install --cask hammerspoon`
2. このフォルダの [`hammerspoon/init.lua`](hammerspoon/init.lua) を `~/.hammerspoon/init.lua` にコピー
3. Hammerspoon を起動し、アクセシビリティ権限を許可
4. 音声入力アプリ側のホットキーを `init.lua` の設定に合わせる(下記「音声入力アプリ」参照)
5. システム設定 → サウンド → 入力 で **Shokz をマイクに選択**

これで Shokz のボタンを押すと音声入力が始まり、もう一度押すと確定します。

### Typeless を使う場合の設定(おすすめ)

[Typeless](https://www.typeless.com/) は AI 整形つきの高精度な音声入力アプリです。方法A と組み合わせる手順:

1. Typeless をインストールして起動(初回のマイク・アクセシビリティ権限を許可)
2. Typeless の **設定 → キーボードショートカット** を開き、「音声入力」のショートカットを **`⌃⌥D`(Control + Option + D)** に変更(Typeless は最大3キーまで)
   - ※ `⌘D` はブラウザの「ブックマーク追加」、`⌥⌘D` は「Dock の表示/非表示」、`⌃⌘D` は「辞書で調べる」と衝突するため避けます
3. Typeless は**デフォルトでトグル動作**(1回押すと開始 / もう1回で停止)なので、特別なモード切替は不要
   - ※ Shokz のボタンは「1回押す」信号しか出せないので、押しっぱなしで使う「Push-to-Talk」は Shokz では使えません(トグルでそのまま動きます)
   - ※「押している間だけ」を Shokz でやりたい場合は **方法B** を参照
4. `init.lua` は初期状態(`MODE = "hotkey"`、`HOTKEY = ⌃⌥D`)のままで OK
5. システム設定 → サウンド → 入力 で **Shokz をマイクに選択**

→ Shokz のボタンを押す → Typeless が録音開始 → 話す → もう一度ボタンを押す → 文字がカーソル位置(Claude / ChatGPT のチャット欄など)に入力され、**約4秒後に自動で Enter が押されて AI に送信**されます(`init.lua` の `AUTO_SEND` 機能)。

### 運用のコツ(離れた場所から使うための前提)

- **AI のチャット欄にカーソルを置いたまま離れる**こと。文字は「いまフォーカスされている入力欄」に入るので、別のウィンドウが前面にあるとそちらに入ってしまいます。
- **自動送信のタイミング**: 長く話すと文字起こしが4秒で終わらず、途中で送信されることがあります。その場合は `init.lua` の `AUTO_SEND_DELAY` を増やしてください。自分で Enter を押したい場合は `AUTO_SEND = false` に。
- **常用するなら自動起動を ON に**: Hammerspoon(環境設定 → Launch Hammerspoon at login)と Typeless のログイン時起動を有効にしておくと、Mac 再起動後も手間なく使えます。
- **AI の返事は画面で読む方式**です。返事も音声で聞きたい(完全に画面を見ずに会話したい)場合は、ChatGPT / Claude アプリの音声モード(方法C)が適しています。

### トラブルシューティング: ボタンを押すと Siri が起動してしまう

OpenComm など **Shokz アプリ非対応モデル**はボタン割り当てを変更できないため、Mac 側で対処します。

**原因**: マルチファンクションボタン(MFB)を**長押し(約2秒)すると音声アシスタント(Siri)が起動**します。この長押し信号は Bluetooth(HFP)の専用コマンドとして OS の奥で処理されるため、Hammerspoon では横取りできません。

**対処(上から順に試す)**:

1. **MFB を「カチッと一瞬だけ」短押しする**。短押しは再生/一時停止(`PLAY`)として届くので Hammerspoon で拾えます。長く押さないのがコツ。
2. それでも Siri が出る/反応しないなら、**引き金を音量ボタンに変える**。`init.lua` の先頭 `local TRIGGER = "PLAY"` を `"SOUND_UP"`(音量+)または `"SOUND_DOWN"`(音量−)に変更してリロード。音量ボタンは Siri を絶対に呼ばないので確実です(その代わり、選んだ音量ボタンでの音量調整は無効になります)。
3. どの操作がどの信号を送るか不明なときは、[`hammerspoon/diagnose.lua`](hammerspoon/diagnose.lua) を一時的に `~/.hammerspoon/init.lua` に入れて Reload し、ボタンを押して画面表示を確認(`systemKey = PLAY` 等が出れば拾える信号です)。

また、ボタンの信号は「最後に音楽を再生したデバイス」に飛びます。iPhone と Mac に同時接続している場合は、**Mac 側で一度音楽を数秒再生して止めてから**試すと、Mac に信号が向きます。

## 方法B: 「押している間だけ話す」を実現 — 安い Bluetooth ボタンを使う(Mac)

完全に希望どおりの「ボタンを押している間だけ音声入力」(Push-to-Talk)をやるなら、Bluetooth ボタンを1個用意するのが確実です。

| デバイス | 価格目安 | 備考 |
|---|---|---|
| Bluetooth カメラシャッターリモコン (AB Shutter 3 など) | 500〜1,000円 | 「音量+キー」として認識される。Karabiner で別キーに変換 |
| 8BitDo Micro / Zero 2 | 約3,000円 | キーボードモードあり。複数キーを送れる |
| Flic 2 | 約4,000円 | 専用 Mac アプリで押す/長押しに直接アクションを割当可 |

### 仕組み

1. ボタンを Mac とペアリング(キーボードとして認識される)
2. [Karabiner-Elements](https://karabiner-elements.pqrs.org/)(無料)で、ボタンが送るキー(例: 音量+)を `F13` などに変換
3. 音声入力アプリ(Superwhisper / Wispr Flow / VoiceInk)の **ホールドモード(押している間だけ録音)** のホットキーに `F13` を設定
4. マイク入力を Shokz に設定

→ ポケットの中のボタンを**押している間だけ** Shokz のマイクで話せて、離すと文字が入力されます。Bluetooth の届く範囲(約10m)ならどこからでも OK。

## 方法C: iPhone の場合

iPhone は実はボタンすら不要にできます。

- **ChatGPT / Claude アプリの音声モード**: 一度開始すればハンズフリーで会話継続。Shokz のマイクがそのまま使われる
- **アクションボタン**(iPhone 15 Pro 以降): ショートカット経由で「ChatGPT 音声モードを開く」などを割当
- **背面タップ**(設定 → アクセシビリティ → タッチ → 背面タップ): iPhone の背面を2回叩くとショートカット起動
- キーボードの音声入力もマイクボタン1タップで開始でき、Shokz のマイクが使われる

## 音声入力アプリの選択肢(Mac)

文字起こしして AI のチャット欄に入力するためのアプリです。どれも「カーソルがある場所」にテキストが入るので、Claude や ChatGPT のウィンドウを開いておけばそこに入ります。

| アプリ | 価格 | ホールド(押す間だけ)対応 | 備考 |
|---|---|---|---|
| macOS 標準の音声入力 | 無料 | ✕(トグルのみ) | 設定 → キーボード → 音声入力 で有効化 |
| [VoiceInk](https://tryvoiceink.com/) | 無料(OSS) | ○ | ローカル Whisper で高精度。日本語OK |
| [Superwhisper](https://superwhisper.com/) | 無料枠あり | ○ | 人気。AIによる整形機能つき |
| [Wispr Flow](https://wisprflow.ai/) | 有料 | ○ | 精度・速度とも良い |

## 複数の Mac で使う(Mac mini と MacBook Air など)

設定はクラウド同期されないため、**使いたい Mac ごとに同じセットアップを1回ずつ**行います。

1. そのMacのターミナルでクイックスタートの1行(install.sh)を実行
2. Hammerspoon のアクセシビリティ権限を許可 → 🔨 → Reload Config
3. Typeless をインストールして**同じアカウントでログイン**し、ショートカットを `⌃⌥D` に設定(全Macで揃える)
4. マイク入力を Shokz にする

### 2台運用の注意

- **Shokz・ボタンの接続先**: Bluetooth の音声とボタン信号は「いまアクティブに使っているデバイス」に向かいます。使う Mac を切り替えるときは、**その Mac で音を一瞬再生して接続を引き寄せる**か、使わない Mac の Bluetooth から Shokz を「切断」しておくと確実です。
- 2台が同時に Shokz を掴もうとすると取り合いになるため、**「いま使う1台にだけ接続」**が安定します。
- 8BitDo / Flic 2 などの引き金ボタンも、ペアリングは基本1台ずつ。Mac を移るときはつなぎ直してください。

---

## 注意点

- **Shokz を Mac に接続しておくこと**: マルチポイント対応モデル(OpenComm, OpenRun Pro など)なら iPhone と Mac に同時接続できます。マルチファンクションボタンの信号は「最後に再生操作したデバイス」に飛ぶので、方法Aを使うときは Mac 側で使う想定にしてください。
- **マイク使用中は音質が落ちる**: Bluetooth の仕様(HFP プロファイル)上、マイクを使っている間は再生音質がモノラル・低音質になります。音声入力をやめると元に戻ります。
- **届く距離は Bluetooth の範囲(約10m、壁があると短くなる)**。家全体をカバーしたい場合は iPhone 側(方法C)と併用するのがおすすめ。
